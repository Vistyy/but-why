# Keep v1 PR heads locally owned

V1 supports one local Change, one local branch, and one non-draft PR whose head is published only from an exact validated local Candidate.
An external PR head or base change becomes Needs Input instead of being adopted automatically, while later vertical slices may add remote Candidate capture after its provenance and ownership rules are designed.
This keeps publication and reconciliation safe without making initial delivery depend on forks, remote-only commits, or unclear branch authority.
