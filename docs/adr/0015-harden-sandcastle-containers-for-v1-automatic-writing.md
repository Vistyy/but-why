# Harden Sandcastle containers for v1 automatic writing

V1 automatic code-writing uses Sandcastle's built-in Docker or rootless Podman provider with a fixed image, non-root user, explicit environment allowlist, no GitHub credentials, no extra mounts, supplementary groups, host devices, or Docker socket, bounded untrusted evidence, and enforced resource and time limits where the provider supports them.
Pi cannot push; But Why? captures and fully validates the successor Candidate, rechecks the exact remote SHA, and performs the parent-controlled compare-and-swap push while a human remains responsible for merge.
This is an explicit opt-in security trade-off because ordinary containers share the host kernel and lack built-in credential proxying and deny-by-default domain egress; a stronger OpenShell or Gondolin isolated Sandcastle provider remains a later security improvement.
